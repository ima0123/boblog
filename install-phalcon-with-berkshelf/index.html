爆速Phalconの環境をBerkshelfで爆速インストール

前回Chef-soloのcookbookを手動でgit cloneする形でインストールしていたのですが
Berkshelfを使うとその手間も省けて、より爆速に環境がインストールできますね。

今回はまっさらなvagrantが既に vagrant upされている状態という前提から
説明をスタートします。

Chefリポジトリの作成

まずはChefリポジトリを作成します。

                
                  knife solo init ./chef-repository

作成されたフォルダの中に Berksfileというファイルができているので編集します。

                
                  cd chef-repository
                  vi
                  Berksfile 

                  // 以下を追記します # cookbook
                  "phalcon", git:
                  "https://github.com/aiMatz/cookbook-phalcon.git",
                  branch: 
                  "master"

Berkshelfでcookbookをダウンロード

Berksfileと同じディレクトリで以下のコマンドを実行します。

                
                  berks vendor

すると、berks-cookbooksというフォルダが作成され、その中にphalconフォルダが作成されます。
この中身は、githubからダウンロードしたcookbookになるわけですね。

続けて、 node/vagrant.localhostのrun_listに recipe[phalcon]を追加します。

knife-soloの実行

現状、まだvagrantのBOXには knife solo prepare vagrantのコマンドを実行していないため
Chefがインストールされていない状態です。

Chefがインストールされていない状態でも実行するためには、

                
                  knife solo
                  bootstrap vagrant

を実行すると、一時的にChefのインストールとCookbookの実行を同時に行ってくれるので便利です。
ただ、再度 knife solo cook vagrantをしようとすると、Chefがインストールされていないと怒られるので、
予め、 knife solo prepare vagrantを実行しておくことをおすすめします。
