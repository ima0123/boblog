超高速PHPフレームワークPhalconを使ったWeb制作

Falconと間違えやすいですよね、Phalcon。
インストール時にこの勘違いは致命的です。。

http://phalconphp.com/ja/

爆速とよばれるPHPフレームワークPhalcon

最近Phalconを使ってWeb制作をしていますが非常に高速に動作し、
仕組みもいわゆるフレームワークの概念から大きく外れておらず
開発のやりがいのある印象です。

Phalconがどれだけ早いかというのは、以下のサイトにベンチマークが掲載されています。

[https://docs.phalconphp.com/ja/latest/reference/benchmark/hello-world.html:title]

（2015/08/04 リンク更新）

ちゃんと、前提条件を明記してくれていたベンチで信頼性が高めですね。
手元でも似たようなベンチ結果が出ました。

Phalconを使うようになった理由はいくつかあります。

  • Core技術がC言語で書かれているが故、（他のPHPフレームワークと比較して）高速に動作する。
  • 今後の広がりがある程度見えること（廃れていくニッチなフレームワークにはならなさそう）

困っていることといえば、

  • 日本語翻訳チームに参加を申し込んだのですが、もうメンバーいっぱいなんだよと断られたにもかかわらず
    日本語ドキュメントがなかなか整理されない現状 ・・・orz

日本にももっと広めたいのに。。

（そして日本語ドキュメントは整理されていない現状orz）

ということもあり、自分のブログで紹介することで少しでもPhalconが認知され、
少しでも広く使われるようになることを期待して記事を書くことにしました。

では、早速インストール！

インストール

OS: Linux (CentOS release 6.5 (Final)) ※vagrantを使用したMac上のローカル環境

PHPのインストール

PhalconはPHP 5.3.6以上推奨の模様。
CentOSのbaseはこれよりも低い可能性があるため、remiからダウンロードします。

必要なパッケージ類のインストールのために
remi, epel, rpm-forgeレポジトリを追加します。

サードパーティ製のリポジトリを追加するので、優先度をつけるyum-propertiesを導入する。
(詳細はこちらを参照させていただきました)

yum install yum-plugin-priorities

# epel
$ rpm -Uvh http://ftp-srv2.kddilabs.jp/Linux/distributions/fedora/epel/6/x86_64/epel-release-6-8.noarch.rpm

# remi
$ rpm -Uvh http://rpms.famillecollet.com/enterprise/remi-release-6.rpm

# rpmforge
$ rpm -Uvh http://pkgs.repoforge.org/rpmforge-release/rpmforge-release-0.5.3-1.el6.rf.x86_64.rpm

必要なパッケージ類のインストール

yum install enablerepo=remi php-devel php-mysql gcc libtool git

php -vしてバージョンを確認。

phalconのダウンロード&インストール

cd /usr/local/src/
git clone git://github.com/phalcon/cphalcon.git
cd cphalcon/build
./install #←別ディレクトリからの実行はエラーがでます

PHPの設定

PHPモジュールにPhalconの動的ライブラリを読み込ませます。

$ vi /etc/php.d/phalcon.ini
extension = phalcon.so

HTTPサーバー再起動
service httpd restart

Phalcon Developer Toolsのインストール

Composerからインストールする

cd /usr/local/shareなどで

curl -s http://getcomposer.org/installer | php

同階層でcomposer.jsonを作成

{
    "require": {
        "phalcon/devtools": "dev-master"
    }
}

composer installを実行

php composer.phar install

cd /vender/phalcon/devtools以下のphalcon.shを実行

ln -s ~/devtools/phalcon.php /usr/bin/phalcon
chmod ugo+x /usr/bin/phalcon

phalconとコマンドを打って、

# phalcon

Phalcon DevTools (1.2.4)

Available commands:
  commands (alias of: list, enumerate)
  controller (alias of: create-controller)
  model (alias of: create-model)
  all-models (alias of: create-all-models)
  project (alias of: create-project)
  scaffold
  migration
  webtools

と表示されればPhalconのインストールは完了！

爆速PhalconをChef-soloで爆速インストール（Vagrant編）

コア部分がC言語で記述されているがゆえ爆速に動作するPhalcon。
これを爆速で動作できる環境をつくるために、Chefレシピを用意しました。

https://github.com/aiMatz/cookbook-phalcon

Phalconを動かすまで意外と道のりがありまして、
非常に手間なのでChefでやるのがベターですね。

ここ最近は、vagrantが出てきたこともあって、
開発マシンを汚さずに手軽に作成することができて、便利な世の中です。

vagrantへの各種インストール作業は、Chefのcookbookを使います。
実行はやっぱり、knife-soloですよね。

vagrantの用意

vagrantのインストールは、ググってくだされ(・ω・)

vagrant box（OSイメージ）の取得

Boxイメージは何でもOKですが、CentOS 6.5を使用してみた。

// Macのターミナル
vagrant box add "CentOS 6.5 x86_64" https://github.com/2creatives/vagrant-centos/releases/download/v6.5.3/centos65-x86_64-20140116.box


vagrant init "CentOS 6.5 x86_64"

// vagrant起動！
vagrant up


// vagrantにSSH接続ができるように設定
vagrant ssh-config --host vagrant >> ~/.ssh/config

// vagrantに接続できることを確認
ssh vagrant

exit

※boxのイメージは、http://www.vagrantbox.es/とかにたくさんあります。

knife-soloのインストール

knife-soloをインストールすると一式がインストールされるかと

// Macのターミナル
gem i knife-solo

chef cookbookの準備

knife-soloでリポジトリを作成

// Macのターミナル
knife solo init chef-repository

phalconインストール用のcookbookをダウンロード

// Macのターミナル
git clone https://github.com/aiMatz/cookbook-phalcon chef-repository/site-cookbooks/phalcon

cd chef-repository

ここでvagrantのイメージにChefをインストール。

// Macのターミナル
knife solo prepare vagrant

実行後、node/default.rbが作成されます。
recipe[phalcon]をrun_listに追加します。

// Macのターミナル
vi node/default.rb

# {"run_list": [
    "recipe[phalcon]"
    ]
  }

準備ができたので、料理開始！！

// Macのターミナル
knife solo cook vagrant

諸々のインストールが進んでいくはずです。
完了したら、SSHログインして

phalcon

とコマンドを打って、phalcon devtoolsの説明が表示されればインストールが完了です。

Enjoy Phalcon!!

爆速Phalconの環境をBerkshelfで爆速インストール

前回Chef-soloのcookbookを手動でgit cloneする形でインストールしていたのですが
Berkshelfを使うとその手間も省けて、より爆速に環境がインストールできますね。

今回はまっさらなvagrantが既にvagrant upされている状態という前提から
説明をスタートします。

Chefリポジトリの作成

まずはChefリポジトリを作成します。

knife solo init ./chef-repository

作成されたフォルダの中にBerksfileというファイルができているので編集します。

cd chef-repository
vi Berksfile 

// 以下を追記します
# cookbook "phalcon", git: "https://github.com/aiMatz/cookbook-phalcon.git", branch: "master"

Berkshelfでcookbookをダウンロード

Berksfileと同じディレクトリで以下のコマンドを実行します。

berks vendor

すると、berks-cookbooksというフォルダが作成され、その中にphalconフォルダが作成されます。
この中身は、githubからダウンロードしたcookbookになるわけですね。

続けて、node/vagrant.localhostのrun_listにrecipe[phalcon]を追加します。

knife-soloの実行

現状、まだvagrantのBOXにはknife solo prepare vagrantのコマンドを実行していないため
Chefがインストールされていない状態です。

Chefがインストールされていない状態でも実行するためには、

knife solo bootstrap vagrant

を実行すると、一時的にChefのインストールとCookbookの実行を同時に行ってくれるので便利です。
ただ、再度knife solo cook vagrantをしようとすると、Chefがインストールされていないと怒られるので、
予め、knife solo prepare vagrantを実行しておくことをおすすめします。

Mac御用達のParallelsDesktopを使ってVagrantを起動する

ParallelsDesktopとVirtualBoxの起動の両立ができない

僕はMacユーザーなのですが、WindowsPCを使う機会も少々あるので
ParallelsDesktopというソフトを使ってMacの上でWindows7を起動して使っています。

ParallelsDesktopを使っていて不便なのは
ParallelsDesktopを起動している状態でVagrant(VirtualBox)を起動をするとエラーが発生していまうことです。

今まではVagrant(VirtualBox)を使って起動していた仮想OSを全て停止して
ParallelsDesktopを起動したり、その逆もしかり、という状態で非常に不便。。

そこでVagrantをParallelsDesktop経由で起動するように設定しました。

VagrantをVirtualBoxではなくParallelsDesktopで行う

WindowsをVirtualBoxで起動する方法やVirtualBoxとParallelsDesktopを共存させる方法なども検討しましたが
VirtualBoxを使っていたVagrantをParalellsDesktopに移動させることにしました。

メリット

  • VirtualBoxがもういらない

  • Vagrantで起動したCentOSを起動した状態で、ParallelsDesktopでWindowsを起動できる
